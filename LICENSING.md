## Why this license?

Civic.Social is built on the principle that civic infrastructure should be **open and shared**. The protocols and specifications — Civic Hub, Civic Event, Civic Process, Civic Identity — are published under [Creative Commons Attribution 4.0](https://github.com/creatinglake/civic-social-docs) so that anyone can build conforming implementations.

The reference implementations themselves — including this software — are licensed differently. We chose BUSL because:

1. **It keeps the source readable** so people can verify, learn from, and contribute to the implementation.
2. **It reserves commercial deployment rights** so that revenue from production use can be reinvested in maintaining the commons rather than enriching shareholders.
3. **It guarantees eventual openness** — every release becomes Apache 2.0 within four years, automatically and irrevocably.

This pattern is used by HashiCorp (Terraform), MariaDB, Sentry, CockroachDB, and others.

## Building your own implementation

If you want to build your own civic hub implementation rather than use this one, that's encouraged. The [Civic Hub specification](https://github.com/creatinglake/civic-social-docs/blob/main/ecosystem/civic-hub-spec.md) is published under CC-BY 4.0. Anyone can implement it under any license they choose — including commercial, closed-source, or open-source licenses. The spec license does not constrain implementations.
